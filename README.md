# 台股策略回測與績效分析系統

## 一、專題簡介

本專題以台股 KBar 資料為基礎，將課堂提供的回測程式整理成一個可操作的 Streamlit 網頁系統。使用者可以在網頁中選擇回測期間、KBar 週期、交易數量與停損設定，並比較不同技術指標策略的交易績效。

本系統除了完成原本的移動平均線、RSI 與布林通道策略，也另外補上作業要求的 MACD 策略與 KDJ 策略。為了讓策略比較更完整，系統也加入參數最佳化、風險調整績效指標、交易紀錄下載，以及生成式 AI 績效分析與追問功能。

## 三、系統功能

本系統主要功能如下：

- 可選擇回測日期區間。
- 可切換 KBar 週期：1 分、5 分、15 分、30 分、60 分、日 K。
- 可設定每次交易數量與移動停損點數。
- 可同時比較多個策略的回測結果。
- 顯示累積損益曲線與每筆交易紀錄。
- 提供策略績效比較表。
- 提供各策略參數最佳化功能。
- 提供最佳參數彙整表。
- 可下載績效比較、交易紀錄、價格資料與最佳化結果。
- 可使用 DeepSeek API 進行策略績效自動分析。
- 可針對目前回測結果進行即時追問。

## 四、策略設計

本專題實作以下六種策略：

| 策略 | 主要邏輯 |
| --- | --- |
| 移動平均線策略 | 短期均線向上突破長期均線時做多，向下跌破時做空或反向。 |
| RSI 順勢策略 | 短週期 RSI 向上突破長週期 RSI 視為多方動能增強，向下跌破視為空方動能增強。 |
| RSI 逆勢策略 | RSI 從低檔區向上突破視為超賣反彈，從高檔區向下跌破視為超買回落。 |
| 布林通道策略 | 價格突破上軌視為強勢訊號，跌破下軌視為弱勢訊號，回到中軌附近出場。 |
| MACD 策略 | DIF 向上突破 DEA 為黃金交叉，向下跌破 DEA 為死亡交叉，並加入零軸濾網。 |
| KDJ 策略 | K 值向上突破 D 值為買進訊號，向下跌破 D 值為賣出訊號，並加入超買超賣區間濾網。 |

## 五、績效指標

為了避免只用總損益判斷策略好壞，本系統同時計算報酬與風險相關指標：

- 總損益
- 交易次數
- 勝率
- 平均每筆損益
- 最大回撤
- Profit Factor
- 報酬回撤比
- 類 Sharpe 指標
- 每筆交易期望值
- 綜合分數

其中綜合分數會同時考慮總損益、勝率、最大回撤與報酬回撤比，用來做為策略排序與參數最佳化的依據。

## 六、參數最佳化

每個策略都有設定一組合理的參數範圍，例如：

- 移動平均線策略：短均線與長均線期間。
- RSI 策略：RSI 期間、上下界門檻。
- 布林通道策略：期間與標準差倍數。
- MACD 策略：快線、慢線與訊號線期間。
- KDJ 策略：RSV、K、D 的平滑期間。

系統會針對不同參數組合進行回測，並依照綜合分數排序，找出各策略表現較佳的參數設定。

## 七、生成式 AI 分析

本系統加入 DeepSeek API 作為生成式 AI 分析工具。AI 會根據目前回測結果與最佳化摘要，自動整理各策略的報酬、風險、勝率、最大回撤與改善方向。

另外也設計了追問功能，使用者可以針對目前回測結果提問，例如：

- 哪個策略比較適合保守投資人？
- MACD 策略為什麼表現比較弱？
- 如果想降低最大回撤，應該調整哪些參數？
- 布林通道策略和 KDJ 策略哪一個比較穩定？

如果沒有設定 API key，系統仍會提供基本的離線規則式績效分析。

## 八、執行方式

本機執行方式：

```bash
pip install -r requirements.txt
streamlit run app.py
```

Streamlit Cloud 部署時，主程式檔案設定為：

```text
app.py
```

如果要啟用 DeepSeek 分析功能，需要在 Streamlit Cloud 的 Secrets 中設定：

```toml
DEEPSEEK_API_KEY = "自己的 DeepSeek API key"
```

## 十、風險提醒

本系統的結果是依據歷史資料進行回測，主要用途為課程作業與策略比較。歷史回測績效不代表未來一定會有相同結果，實際交易仍需考慮交易成本、滑價、流動性與市場風險。
